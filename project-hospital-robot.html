Hospital Sanitation Robot

Autonomous robot using multiple ESP32 microcontrollers, thermal sensing, and ultrasonic sensors to support sanitation workflows.

Project Description

This project explores how low-cost embedded systems can be used to automate parts of hospital sanitation. The robot uses several ESP32 boards connected through a TCA9548A I2C multiplexer to coordinate different tasks such as navigation, scanning, and reporting. The overall behavior is managed by a multi-ESP32 state machine architecture, where a master board controls the high-level states and each slave ESP32 handles its own local actions.

Goals

  • Design a multi-ESP32 architecture for coordinated robot control.
  • Integrate ultrasonic sensors for obstacle detection.
  • Use an MLX90640 thermal array to detect areas of interest.
  • Create a state machine that manages cleaning and scanning modes.

Process

  1. Sketched out the robot states and transitions in a state diagram.
  2. Wired multiple ESP32 boards to the TCA9548A I2C switch.
  3. Implemented individual tasks (movement, sensing, reporting) per board.
  4. Tested each state separately, then integrated the full system.

Tools Used

  • ESP32
  • C / C++
  • FreeRTOS
  • TCA9548A I2C Multiplexer
  • MLX90640 Thermal Sensor
  • Ultrasonic Sensors

Outcomes

The final prototype can navigate a controlled test space, detect objects with ultrasonic sensors, and use thermal data to prioritize regions. The multi-ESP32 design also makes the robot easier to extend with additional modules in the future.
