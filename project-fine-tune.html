Fine-Tune Cell Annotation Tool

An interactive Python tool that lets researchers correct model-generated neuron detections in real time and export clean annotations for retraining.

Project Description

The Fine-Tune Cell Annotation Tool is a desktop application built in Python to help neuroscience labs review and correct model-generated cell detections in Bell Jar, a neurohistological image analysis platform in the Kim Lab. Automated YOLOv8 detections are loaded on top of tiled brain images, and users can directly edit bounding boxes before saving corrected labels for model retraining.

This project addresses a common problem in computer vision workflows: models are good but not perfect, and fully manual annotation is too slow. Fine-Tune creates a feedback loop where user corrections make the model better over time.

Goals

  • Speed up the process of reviewing cell detection predictions.
  • Make it easy for researchers to add, move, and delete bounding boxes.
  • Keep all interaction inside a simple, single-window Python GUI.
  • Export YOLO-format annotations that can be used directly for model fine-tuning.

Methodology

The workflow follows the pipeline from my research poster:

  1. Input image loading: large microscopy images are loaded into the tool.
  2. Tiling: images are split into 640×640 tiles for smooth viewing and editing.
  3. Model prediction: a YOLOv8 model predicts neuron locations on each tile.
  4. User annotation: users add, adjust, or delete bounding boxes with live updates.
  5. Saving annotations: images are saved to an images/ folder and labels to a matching labels/ folder.
  6. Retraining: the cleaned annotations are used to fine-tune the detection model.

Results

Before-and-after comparisons from the poster show that the model performs better after being retrained with user-corrected labels: fewer false positives, fewer missed neurons, and more reliable detections across tiles. The tool also reduces the amount of fully manual annotation time needed for each dataset.

Future Directions

  • Integrate the Fine-Tune workflow directly into the main Bell Jar application.
  • Improve model behavior so it does not miss detections at standard confidence thresholds.
  • Fix fullscreen and window-closing bugs to make the app more robust for long sessions.
  • Add more options for tile management and custom slicing sizes.

Tools Used

  • Python
  • PyQt GUI Toolkit
  • OpenCV
  • YOLOv8
  • Machine Learning

Outcomes

This project produced a complete annotation and feedback pipeline that turns manual edits into structured training data. It also gave me experience with GUI design, image tiling, bounding box logic, and integrating model retraining into a research workflow.
